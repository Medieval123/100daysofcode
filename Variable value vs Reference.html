Variables by values vs Reference.

Passing by Value

  • Given b=a, passing/copying by value means changing copied value in b does not affect the value stored in a and vice versa.

Passing by Reference.

  • Given b=a, passing/copying by Reference means changing copied value in b does indeed affect the value stored in a and vice versa.
  • In Javascript, Primitives are passed by value, while Objects are passed by Reference.
  • "Under the hood", everything is actually passed by value.
  • The difference comes because of the way the Objects and Primitives are stored in memory.

The difference bettween Passed by value and References.

  • In Passed by value, the values, once it is changed do not affect the Original value of the Original variable.
  • As Opposed to Passed by Reference, when yoy change the value, the Original object that it is pointing to changes as well.
